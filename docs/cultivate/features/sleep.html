寝落ち送り

VC で寝落ちしたメンバーを移動。コマンド/右クリック/メンションの3方式に対応

概要

寝落ち送り機能は、VC で寝落ち（長時間の無音・無反応状態）したメンバーを 専用の「寝落ち部屋」VC に移動させる機能です。
v2.0 から複数プリセットが設定できるようになりました。 用途に応じた設定を複数作成し、VC やカテゴリごとに異なる動作を割り当てられます。

3つの寝落ち送り方法

方法 操作 備考
① スラッシュコマンド /寝落ち送り <メンバー> 誰でも実行可能。対象メンバーの参加VCからプリセットを自動判定
② メッセージ右クリック メッセージを右クリック → アプリ → 寝落ち送り メッセージ投稿者を寝落ち送り
③ メンバー右クリック メンバーを右クリック → アプリ → 寝落ち送り 選択したメンバーを寝落ち送り

💡 ヒント: ①③ は Bot が入れない VC や、メッセージ参照のない場面でも使えます。

設定方法

設定パネルを開く

/寝落ち送り設定 を実行すると、プリセット一覧が表示されます。
このコマンドは管理者のみ実行可能です。

プリセット管理

ボタン 動作
➕ 新規作成 新しいプリセットを追加します（最大数制限なし）
✏️ 編集 選択中のプリセットの詳細設定を開きます
🗑️ 削除 任意のプリセットを削除します

プリセット設定項目

設定項目 必須 説明
📛 名前変更 任意 プリセットの表示名を変更します
✅ 有効/無効切替 — このプリセットを使うかどうかを切り替えます
💬 送信チャンネル 必須 従来のメンション方式用。寝落ち送りを受け付けるテキストチャンネル
🔉 送り先VC 必須 寝落ちしたメンバーを移動させるVC
👥 寝落ち送りロール 任意 このロールを持つメンバーのみ寝落ち送り対象になります
🔌 寝落ち切断ロール 任意 このロールを持つメンバーは寝落ち送り時に切断されます
🚫 送信拒否ロール 任意 このロールを持つメンバーは寝落ち送りされません
⚙️ デフォルト動作 任意 ロール条件に合致しないメンバーのデフォルト動作（送り/切断/拒否）
📋 ホワイトリスト 任意 このプリセットの適用対象となるカテゴリ/VC
📋 ブラックリスト 任意 このプリセットの適用除外となるカテゴリ/VC

プリセットの優先順位

ある VC に複数のプリセットがマッチする場合、プリセットIDが最も小さい（=先に作られた）ものが優先されます。

例: ID=1 に「一般VC → 寝落ちVC A」、ID=2 に「一般VC → 寝落ちVC B」と設定した場合、ID=1 が優先され「寝落ちVC A」に送られます。

マッチング条件

対象メンバーの参加VCから、以下の条件で最適なプリセットを探します。

  1. 有効なプリセットのみ 無効なプリセットはスキップされます
  2. 送り先VCが設定されていること voice_channel_id が未設定のものはスキップ
  3. ホワイトリストチェック 設定されている場合、対象VC（またはそのカテゴリ）が含まれなければスキップ
  4. ブラックリストチェック 設定されている場合、対象VC（またはそのカテゴリ）が含まれていればスキップ
  5. ID昇順で最初にマッチしたものを使用

⚠️ 設計上の注意: 重複したカテゴリ/VCにまたがるプリセットを作ることは非推奨です。IDの小さいものが常に優先されることを理解した上で設定してください。

⚠️ ロール優先度: 送信拒否ロール ＞ 寝落ち切断ロール ＞ 寝落ち送りロール ＞ デフォルト動作 の順で判定されます。
