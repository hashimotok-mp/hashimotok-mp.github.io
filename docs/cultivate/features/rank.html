ランクシステム

チャット・VCの活動を計測してレベルアップ

💡 ランクシステムとは？ メッセージ送信やVC接続時間に応じて経験値（XP）を 獲得し、レベルアップしていくシステムです。 サーバーごとに自由に成長曲線を設計できます。

🎯 設計思想: レベルは常にXPからその場計算されます。 保存されるのは生のXP値のみで、 ランク表示時に設定に応じて加算してレベルを算出します。 そのため、成長曲線を変更すると過去のXPから 新しいレベルが自動的に再計算されます。

概要

ランクシステムは、ユーザーのDiscord活動量を可視化する機能です。 チャットでの発言やVCへの接続時間に応じてXPが溜まり、 一定量貯まるとレベルアップします。

ミュート勢の利便性を考慮し、VCミュート中もXPが獲得できる設計です。 また、テキストチャットとインチャットを別々に計測する設定も可能です。

経験値の獲得方法

チャット経験値

メッセージを送信するとXPが付与されます。 ただし、以下の条件で付与されません。

  • Botユーザーのメッセージ
  • ! や / で始まるコマンド
  • クールダウン期間中の連続送信（デフォルト60秒）
  • 有効期間外（active_start / active_end が設定されている場合）
  • チャンネルフィルターで拒否されているチャンネル

チャットXPの計算フロー

1. 基本XP = _calc_random_exp(chat=True)
   ├── モード0（固定値）: chat_exp × chat_multiplier
   ├── モード1（一様）: random.uniform(min, max)
   ├── モード2（三角）: random.triangular(min, max)
   └── モード3（カスタム）: ユーザー定義式 (lo, hi, R 利用可)

2. + メッセージボーナス
   ├── 文字数ボーナス（設定文字数以上で加算）
   ├── 添付ファイルボーナス（1添付ごとに加算）
   └── スタンプボーナス（1スタンプごとに加算）

3. × イベントブースト倍率（boost_chat_mult）

4. XPキャップ適用（日次上限を超過しないよう調整）

メッセージボーナス

メッセージの質に応じて追加XPが付与されます。 管理者が各項目を個別に設定できます。

ボーナス種別 条件 加算量
📝 文字数ボーナス 設定文字数以上（例: 100文字） 設定値（例: +5 XP）
📎 添付ファイル メッセージにファイル添付 設定値 × 添付数（例: +3 × 2）
🏷️ スタンプ メッセージにスタンプ 設定値 × スタンプ数（例: +2 × 1）

VC経験値

VCに接続している間、一定間隔でXPが付与されます。

  • 1分ごとにXPを獲得
  • 2人以上が接続している場合のみ有効
  • ミュート中でもXPが獲得可能
  • 1人VCでは獲得できません
  • VC CHフィルターで拒否されているVCでは付与されません
  • VC切断時に接続開始時からの経過時間に応じて一括計算されます

⚠️ 注意: VCから切断すると、接続開始時からの経過時間に応じたXPが 一括で計算されて加算されます。

XP集計モード

プリセットごとにXP集計モードを選択できます。 設定パネルの「XP集計モード」をクリックするたびに 3つのモードが循環します。

モード 表示名 チャットXPの記録先 VC XPの記録先 VC/Stage内チャット レベル判定
combined 🔄 全体 chat_exp vc_exp chat_exp chat_exp + vc_exp（合算）
separate 🔀 VC/チャット chat_exp vc_exp chat_exp 各XPから独立（2系統）
inchat 🎙️ VC/チャット/インチャット chat_exp vc_exp inchat_exp（独立） 各XPから独立（3系統）
  • 全体 (combined): chat_exp と vc_exp を 合算して単一レベルを計算します。 チャットとVCの両方の活動が同じレベルに貢献するシンプルな方式です。
  • VC/チャット (separate): チャットXP → chat_exp、 VC XP → vc_exp として別々に記録し、 それぞれから独立したレベルを計算します。 チャットとVCで別々にレベルアップが可能です。
  • VC/チャット/インチャット (inchat): separate と同様にチャット・VCで独立したレベルを持ちつつ、 VCチャンネル内のテキストチャットやステージチャンネルでの 発言のみが別テーブル inchat_exp に記録され、 3つ目の独立したレベルとして表示されます。

🔑 重要な設計: レベルは保存されず、全ての表示でXPからその場計算されます。 そのため成長曲線の変更が過去のデータに即座に反映され、 レベルがずれる心配はありません。

ランダムEXPシステム

経験値の獲得量にバラつきを持たせられるランダムEXP機能を 搭載しています。4つのモードから選択可能です。

モード 名称 挙動 用途
0 固定値 常に一定値（従来の設定値 × 倍率） 安定したXP配分
1 一様ランダム random.uniform(最小, 最大) シンプルなランダム
2 三角分布 random.triangular(最小, 最大) 中央値付近が出やすい
3 カスタム数式 ユーザー定義の式で計算 自由な設計

カスタム数式モード（モード3）

以下の変数が利用可能です：

変数 説明
lo 設定されている下限XP
hi 設定されている上限XP
R random.random() — 0～1のランダム値

設定例：

# デフォルト（一様ランダムと同じ挙動）
lo + int(R * (hi - lo + 1))

# 正規分布風（Box-Muller法の簡易版）
int((lo + hi) / 2 + (R - 0.5) * (hi - lo))

# 高確率で最大値近く
int(hi * R ** 0.5)

拡張機能

📅 XPキャップ

1日・1週間あたりの獲得XPに上限を設定できます。 キャップを超えたXPは切り捨てられます。

  • 日次上限: その日の0時からの合計獲得XPを制限
  • 週次上限: 月曜0時からの合計獲得XPを制限
  • 0を設定すると上限なし（デフォルト）

🚀 イベントブースト

特定の期間、XP獲得量に倍率を掛けられるイベント機能です。

  • チャット倍率 / VC倍率を個別に設定
  • 開始日時・終了日時を設定（期間限定ブースト）
  • 期間外や無効時は倍率1.0x（通常状態）

🟢 有効期間

ランクシステムの計測自体を特定の期間だけ有効にできます。

  • 開始日時のみ設定 → その日時以降のみ計測
  • 終了日時のみ設定 → その日時までのみ計測
  • 両方空欄 → 常時有効（デフォルト）
  • 有効期間外は一切XPが付与されません

#️⃣ チャンネルフィルター

特定のチャンネル・カテゴリをXP計測の対象・除外に指定できます。 テキストCH（TC）とVCで独立して設定可能です。

フィルターはホワイトリスト（許可）と ブラックリスト（拒否）の2つを 同時に設定できます。

リスト 挙動
✅ ホワイトリスト 設定されている場合、該当CH/カテゴリのみXP計測。 未設定（空）の場合は全CHが対象。
❌ ブラックリスト 該当CH/カテゴリをXP計測から除外。 ホワイトリストより優先（ホワイトにあってもBLなら除外）。

テキストCH（TC）とVCでそれぞれ以下の4項目を設定できます：

  • ✅ カテゴリWH（そのカテゴリ内の全CHが許可対象）
  • ✅ チャンネルWH（個別のCHを許可対象）
  • ❌ カテゴリBL（そのカテゴリ内の全CHを除外）
  • ❌ チャンネルBL（個別のCHを除外）

設定は専用Modalで行います。Modal内の4つのChannelSelectで それぞれWH/BLのカテゴリ・チャンネルを選択できます。

📊 期間ランキング

/ranking コマンドに period オプションを追加することで、 週間・月間のランキングを表示できます。

  • 週間: ISO週単位（月曜～日曜）
  • 月間: 暦月単位（1日～月末）
  • 指定なし: 全期間の累計ランキング

高度設定：カスタム数式

管理者は任意の数式で成長曲線を設計できます。 テンプレートにない独自の曲線も自由に設定可能です。

使用可能な構文

要素 説明 例
L 現在のレベル変数 5 * L + 100
+ - * / // % ** 四則演算・切り捨て除算・剰余・累乗 L ** 2
min max abs 最小・最大・絶対値 max(100, 50 * L)
pow int float round 累乗・整数化・浮動小数点化・丸め pow(L, 2.5)

設定例

# デフォルト（Easy Start）
5 * L ** 2 + 50 * L

# Standard（旧デフォルト）
5 * L ** 2 + 50 * L + 100

# 線形制（ゲームバランス向け）
100 * L + 50

# 緩やか→急激（イベント向け）
max(100, 50 * L ** 1.5)

# 超長期サーバー向け（Lv.100↑が目標）
200 * L ** 1.8

💡 累積XP式について: 累積XPの計算式は通常、次レベルXP式から自動導出されます。 高度な用途で独自に設定したい場合のみ、累積XP式も入力できます。

⭐ メイン設定

複数のプリセットを作成できるランクシステムでは、 メイン設定機能で /rank コマンドの表示対象を1つに絞れます。

💡 仕組み:

  • 各プリセットのエディタ項目から「⭐ メイン設定」をONにできます
  • メインをONにすると、他の全プリセットのメイン設定が自動的にOFFになります
  • /rank コマンドはメイン設定されたプリセットのみ表示します
  • メイン設定が1つもない場合、/rank はエラーメッセージを表示します
  • 設定パネル上ではメイン設定のプリセットに ⭐ アイコンが表示されます
  • /ranking やXP計測はメイン設定に関係なく全プリセットが動作します

📋 ログ設定

ランクシステムの各イベントを特定のチャンネルにログ出力できます。 5つのカテゴリに分かれており、それぞれ個別に 出力先チャンネルを設定可能です。 設定されていないカテゴリはグローバル設定にフォールバックします。

設定方法

  1. /rank設定 を実行
  2. プリセットのエディタを開く
  3. 左のSelectで「📋 ログ設定」を選択
  4. ログ設定Viewでカテゴリを選択
  5. Modal内のChannelSelectで出力先チャンネルを選択（テキストCH / 公開スレッド / プライベートスレッドから選択可能）
  6. 未設定のカテゴリはグローバルCHにフォールバックします
  7. 全て未設定の場合はログは出力されません

Modal内のChannelSelectは min_values=0 に設定されているため、 選択を全て解除（クリア）することでそのカテゴリの設定を無効化できます。

注意事項

  • BotユーザーのメッセージはXP計測の対象外です
  • 同じメッセージの連投はクールダウンで制限されます
  • VCは2人以上で接続している場合のみXPが付与されます
  • レベルアップ通知は設定したチャンネルに送信されます
  • カスタム数式の結果が0以下になった場合、デフォルト式に戻ります
  • XPキャップは日次リセット（0時）・週次リセット（月曜0時）でリセットされます
  • CHフィルターはCH ID・カテゴリIDの数値で指定します（キーワードでは指定できません）
