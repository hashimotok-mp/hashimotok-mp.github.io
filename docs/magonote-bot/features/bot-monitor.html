Bot監視設定

24時間稼働Botのメッセージを監視し、停止を検知して通知

概要

Bot監視設定機能は、24時間稼働している音楽Botなどのメッセージを監視し、 一定時間メッセージがなかった場合に指定チャンネルへ通知を送信するシステムです。 複数の監視設定を作成でき、それぞれ異なるBotや監視条件を設定できます。

💡 ユースケース: 24時間稼働の音楽Botや各種ユーティリティBotのダウンをいち早く検知し、迅速な対応を可能にします。

使い方

コマンド

/bot監視設定

管理者専用コマンドです。監視設定の一覧が表示され、新規作成・編集・削除が行えます。

監視設定の項目

項目 説明
設定名 監視設定の名前（管理用）
監視対象Bot 監視するBotメンバーを選択
監視チャンネル Botのメッセージを監視するチャンネル
タイムアウト 何分メッセージがなければ通知するか（分）
通知メッセージ 通知時に送信するメッセージ（カスタム可能）
通知先チャンネル 通知を送信するチャンネル
有効/無効 監視のON/OFF

編集パネルのボタン

ボタン 機能
⚙️ 設定 基本設定（設定名・タイムアウト・通知メッセージ・監視チャンネル・通知先チャンネル）をモーダルで編集
🤖 Bot選択 監視対象のBotをセレクトメニューから選択（最大50台対応）
▶️ ON/OFF 監視の有効/無効を切り替え
🗑️ 削除 一覧画面から監視設定を削除
➕ 新規作成 一覧画面で新しい監視設定を作成
💾 保存 編集した内容を保存

仕組み

監視を有効にすると、Botは定期的（約30秒間隔）に監視対象チャンネルの 最新メッセージを確認します。監視対象Botの最終メッセージから 設定したタイムアウト時間が経過している場合、通知先チャンネルに アラートを送信します。通知は同じ監視IDに対して最低5分間隔で送信され、 過剰な通知を防止します。

注意事項

  • 監視には対象Botがメッセージを送信できるチャンネルを指定してください
  • 監視対象Bot・監視チャンネル・通知先チャンネルの全てが設定されていないと監視を有効にできません
  • データベース（MySQL）への接続が必要です
  • Botの再起動後も設定は維持されます
