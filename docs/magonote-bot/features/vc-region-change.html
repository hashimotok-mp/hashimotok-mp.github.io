定期VCリージョン変更

VCのリージョンをスケジュールに従って自動変更

概要

定期VCリージョン変更機能は、指定した VC のリージョン（地域）をスケジュールに従って 定期的に自動変更するシステム機能です。複数のリージョンを巡回するように切り替えたり、 特定の時間帯に特定のリージョンに固定するなどの運用が可能です。

この機能はシステム Cog として動作し、Bot 起動時に自動的に読み込まれます。 データベース（MySQL）に設定を保存し、30秒間隔のループでスケジュールをチェックします。

🔐 管理者専用 この機能の設定・管理はサーバー管理者のみ実行できます。

コマンド

/vcリージョン変更設定
VCリージョン変更のスケジュール一覧を表示・管理するパネルを開きます。
管理者

使い方

1. スケジュールを作成

/vcリージョン変更設定 を実行し、「➕ 新規作成」ボタンで新しいスケジュールを作成します。 複数のスケジュールを保存でき、それぞれ独立した設定を持てます。

2. 基本設定をする

スケジュール一覧で「✏️ 編集」を押すと詳細設定パネルが開きます。 「⚙️ 基本設定」ボタンから以下の項目を設定します。

  • 設定名 — スケジュールの名前（管理用）
  • 変更前リージョン — 有効化時に最初に適用するリージョン（任意）
  • 対象カテゴリ — カテゴリ内の全 VC が対象（複数選択可）
  • 対象VC — 個別の VC を指定（複数選択可）

3. 時間エントリを設定

「⏱ 時間設定」ボタンから、変更するリージョンとその継続時間を設定します。 エントリは登録した順番に実行され、最後までいくと先頭に戻ります（循環）。

1
「➕ 追加」でリージョンと継続時間（分）を指定
2
必要に応じて「✏️ 編集」「🗑️ 削除」で調整
3
「🔙 戻る」でメインパネルに戻り「💾 保存」

4. 有効化

「▶️ ON/OFF」ボタンでスケジュールを有効化します。 有効にすると 30秒間隔のループが設定をチェックし、各エントリの継続時間が経過するたびに 次のリージョンへ自動変更されます。

💡 初回有効化時 「変更前リージョン」が設定されている場合、最初にそのリージョンが適用されてからスケジュールが開始されます。

自動実行の仕組み

この機能は Bot 起動時に 30秒間隔 のループタスクを開始します。 ループの処理フローは以下の通りです。

  1. 全ギルドの全スケジュールをチェック
  2. 無効なスケジュールはスキップ
  3. 最終変更時刻からの経過時間がエントリの継続時間を超えているか確認
  4. 超えている場合、次のエントリのリージョンを全対象 VC に適用
  5. 各 VC の編集間は 0.3秒 のインターバル（レート制限対策）
  6. 変更後、設定された通知チャンネルに変更通知を送信（設定されている場合）
  7. 状態をデータベースに保存

通知メッセージ例: 

🔄 VCリージョン変更 (`スケジュール1`)
リージョンを「香港」に変更しました
（エントリ 2/5）
次の変更まで 60分 です

データベース

設定は MySQL の vc_region_schedules テーブルに保存されます。 テーブル構造は以下の通りです。

カラム名 型 説明
guild_id BIGINT サーバーID（主キー）
schedule_id INT スケジュールID（主キー）
settings_json JSON 設定内容（全項目をJSONで保存）
updated_at TIMESTAMP 最終更新日時（自動更新）
