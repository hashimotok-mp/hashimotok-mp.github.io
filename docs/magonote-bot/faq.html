❓ FAQ

よくある質問とその回答をまとめています。

導入について

Q. まごの手 は無料で使えますか？

A. はい、無料で利用できます。オープンソースの自己ホスト型 Bot です。

Q. 導入には何が必要ですか？

A. Python 3.10 以上の実行環境、Discord Bot トークン、サーバーの管理者権限が必要です。

Q. 複数のサーバーで使用できますか？

A. はい、1つの Bot インスタンスで複数のサーバーを管理できます。設定ファイルでサーバーごとの設定が可能です。

機能について

Q. ながらバッチのボタンはカスタマイズできますか？

A. はい、config.yml の nagara セクションでサーバーごとに自由に設定できます。

Q. 自動リアクションにはいくつの絵文字を設定できますか？

A. 設定ファイルで自由に設定できます。上限は特に設けていませんが、Discord の API 制限に準じます。

Q. キーワードロールで複数のキーワードを設定できますか？

A. はい、1つの設定に複数のキーワードを設定できます。全てのキーワードが含まれている場合にロールが付与されます（AND条件）。

Q. VC接続時権限自動追加で特定のVCを除外できますか？

A. はい、disable_vc リストに除外したいVCのIDを指定することで、特定のVCを対象外にできます。

トラブルシューティング

Q. Bot が反応しません

A. 以下の点をご確認ください。

  • Bot がオンラインになっているか
  • コマンドを実行するチャンネルで Bot の権限が適切に設定されているか
  • 必要な権限（メッセージ管理、ロール管理など）が付与されているか
  • スラッシュコマンドが同期されているか（/sync を試す）

Q. スラッシュコマンドが表示されません

A. Bot 起動後、スラッシュコマンドの同期に時間がかかる場合があります。 管理者は /sync コマンドで手動同期を試してください。

Q. エラーが発生した場合の対処法は？

A. コンソールに出力されるエラーログを確認してください。 Bot は各種ログを logs/ ディレクトリに出力します。 解決しない場合は、サポートサーバーまでエラー内容をご報告ください。

開発について

Q. 機能のリクエストはできますか？

A. GitHub の Issue またはサポートサーバーにて機能リクエストを受け付けています。

Q. バグ報告はどこにすればいいですか？

A. GitHub の Issue またはサポートサーバーまでご連絡ください。 可能であれば、発生した状況やエラーログを添付していただけると助かります。

Q. 自分で機能を追加したいです

A. オープンソースですので、自由にフォークして機能追加が可能です。 Pull Request も歓迎しています。
